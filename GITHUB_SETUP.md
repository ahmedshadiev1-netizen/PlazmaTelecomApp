# Настройка GitHub Actions для сборки APK

## Шаг 1: Создайте репозиторий на GitHub

1. Откройте https://github.com и войдите в аккаунт
2. Нажмите кнопку **"+"** в правом верхнем углу → **"New repository"**
3. **Repository name:** `PlazmaTelecomApp`
4. **Описание:** Оставьте пустым или добавьте описание
5. Оставьте **Public** (или Private, если хотите)
6. **НЕ добавляйте** README, .gitignore, license (они уже есть в проекте)
7. Нажмите **"Create repository"**

## Шаг 2: Загрузите код на GitHub

После создания репозитория GitHub покажет инструкции. Выполните в терминале:

```powershell
cd C:\Users\shadiev-an\Documents\LB\PlazmaTelecomApp

# Замените ВАШ-USERNAME на ваш GitHub username
git remote add origin https://github.com/ВАШ-USERNAME/PlazmaTelecomApp.git

# Переименуем ветку в main (если нужно)
git branch -M main

# Загружаем код
git push -u origin main
```

**Важно:** Замените `ВАШ-USERNAME` на ваш реальный GitHub username!

## Шаг 3: Проверьте сборку

1. После загрузки кода GitHub Actions автоматически запустит сборку
2. Зайдите в репозиторий на GitHub
3. Откройте вкладку **"Actions"**
4. Вы увидите запущенную сборку "Build Android APK"
5. Дождитесь завершения (обычно 10-15 минут)

## Шаг 4: Скачайте APK

1. После завершения сборки откройте последний запуск
2. Прокрутите вниз до раздела **"Artifacts"**
3. Нажмите **"app-release"** для скачивания APK
4. APK готов для установки на Android устройства!

## Что дальше?

- Каждый раз при push в `main`/`master` будет автоматически собираться новый APK
- Можно запустить сборку вручную: Actions → Build Android APK → Run workflow

## Устранение проблем

Если сборка упала:
1. Откройте логи сборки в разделе Actions
2. Найдите конкретную ошибку
3. Исправьте проблему и сделайте новый commit

## Преимущества GitHub Actions

✅ **Бесплатно** - неограниченное количество сборок  
✅ **Автоматически** - сборка при каждом push  
✅ **Не требует установки** - все работает в облаке  
✅ **APK доступен** - всегда можно скачать последнюю версию
